Why Enclave exists

Most communication apps centralize control over storage, metadata, and notifications. Enclave flips that model: your infrastructure, your secrets, your policy boundaries.

Signal-style E2EE model
Self-hosted Supabase backend
Dedicated signaling server
TURN relay for reliable calls
Ntfy-based private push
Android 14+ optimized client

System architecture

Android Client

Jetpack Compose app for chat, calls, vault, and interactive shared modules.

Supabase Stack

Auth, Postgres, storage, and realtime behind Kong in Docker.

Signaling Server

Node.js WebSocket relay for call/session negotiation and live events.

Coturn + Ntfy

Reliable media relay and private push event delivery for resilient sessions.

Setup guide (practical path)

  1. 1. Prepare templates

    cp enclave-ui/local.properties.example enclave-ui/local.properties
cp enclave-server/.env.example enclave-server/.env
cp enclave-server/signaling-server/firebase-adminsdk.json.example enclave-server/signaling-server/firebase-adminsdk.json

  2. 2. Bootstrap local stack

    chmod +x setup-local.sh
./setup-local.sh

  3. 3. Fill required Android keys

    Add SDK, Supabase, signaling, TURN, and Ntfy keys in enclave-ui/local.properties.

  4. 4. Build app

    cd enclave-ui
./gradlew assembleDebug

  5. 5. Production hardening

    Configure DNS + Nginx TLS + Coturn + PM2 as documented in SETUP_GUIDE.md.
